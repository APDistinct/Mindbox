По ходу работы за последние лет 5-6 выработался подход - базу создавать руками, чтобы каждый constraint имел своё собственное уникальное, но известное мне заранее, имя. Это бывает полезным, когда БД имеется в нескольких местах(как минимум локально у разработчика и на сервере для работы) и в случае возникновения ошибок, поиска их решения, можно написать скрипт, работающий с разными объектами по именам и эти имена будут одинаковы во всех экземплярах базы. Ну и отдельно скрипт с тестовой начинкой.
